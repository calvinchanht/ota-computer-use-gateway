# Mickey Workspace Context

Mickey is the first local computer-use testing agent workspace for OTA Computer-Use Gateway.

Purpose:

- validate provider-neutral local computer-use tools;
- support manual Human Connector testing through web chat threads;
- prove workspace boundaries, context files, patch proposals, approvals, and command execution before full OTA core exists.

Current client mode:

- Calvin manually talks to ChatGPT Business or another web chat thread;
- the web thread can reason about tasks;
- the local gateway exposes safe tools for workspace-scoped computer use.

Current #11 direction:

- Mickey is now the proof workspace for OpenClaw-like provider chat-thread agents.
- A fresh provider thread should bootstrap into identity, context, continuity, skills, tools, policy, and browser/CDP capability.
- Mickey proves the pattern before Catalyst is anchored as a job-application browser agent.

MVP boundary:

- no provider-specific automation yet;
- no arbitrary unscoped shell;
- browser/CDP capability is scoped through configured profiles and target ids;
- no destructive or sensitive external actions without Calvin;
- tool use must remain workspace-scoped and audited.
